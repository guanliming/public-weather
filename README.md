# public-weather - 中国天气网爬虫

## 项目简介
本项目是一个基于Python的天气数据爬虫，专门用于从中国天气网（http://www.weather.com.cn）爬取全国各省城市的天气数据。

## 功能特性
- 📊 **全面的数据覆盖**：爬取全国各省城市的天气数据
- 📅 **多时间维度**：包含过去7天的历史天气和未来3天的天气预报
- 💾 **数据持久化**：自动保存为CSV格式，方便后续分析
- 🔄 **中断继续**：支持爬取过程中断后，下次从断点继续
- ⏱️ **友好爬取**：内置100ms爬取间隔，避免对目标网站造成压力
- 📝 **详细注释**：代码包含详细注释，便于理解和维护

## 技术栈
- **Python 3.12**：基于WSL环境的Python 3.12版本
- **requests**：用于发送HTTP请求获取页面内容
- **BeautifulSoup4**：用于解析HTML页面结构
- **pandas**：用于数据处理和CSV文件保存
- **lxml**：高性能XML/HTML解析器

## 使用方法

### 基本运行
在WSL环境中执行：

```bash
# 使用指定的Python环境运行爬虫
/home/dawn/miniconda3/envs/temp312/bin/python weather_crawler.py
```

### 运行流程
1. **加载进度**：检查是否存在进度文件，如果存在则从中断处继续
2. **获取城市列表**：从中国天气网获取全国各省的城市列表
3. **爬取天气数据**：逐个城市爬取天气数据，包含：
   - 未来3天的天气预报（今天、明天、后天）
   - （可选）过去7天的历史天气
4. **保存进度**：每爬取10个城市自动保存一次进度
5. **保存数据**：爬取完成后将所有数据保存为CSV文件

### 中断和继续
- **中断**：在爬取过程中按 `Ctrl+C` 可以安全中断
- **继续**：下次运行时会自动检测进度文件，从断点处继续

### 输出文件说明
爬虫会在 `output` 目录下生成以下文件：

- **部分保存文件**：`weather_data_partial_YYYYMMDD_HHMMSS.csv`
  - 每爬取10个城市自动保存一次
  - 用于防止数据丢失

- **最终保存文件**：`weather_data_final_YYYYMMDD_HHMMSS.csv`
  - 所有城市爬取完成后生成
  - 包含完整的天气数据

### CSV数据字段说明
生成的CSV文件包含以下字段：

| 字段名 | 说明 | 示例 |

| 最低温度 | 当日最低温度 | 15℃ |
| 风向 | 风向 | 东北风 / 西南风 |

## 核心模块说明

### WeatherCrawler 类
主爬虫类，包含所有核心功能：

#### 主要方法：
1. **`__init__()`**：初始化爬虫，设置参数和会话
2. **`load_progress()`**：加载爬取进度，支持中断继续
3. **`save_progress()`**：保存当前爬取进度
4. **`get_city_list()`**：从中国天气网获取全国城市列表

6. **`parse_weather_page()`**：解析天气页面HTML，提取数据
7. **`get_weekday()`**：获取日期对应的星期几
8. **`save_to_csv()`**：将天气数据保存为CSV文件
9. **`run()`**：运行爬虫主流程

### 数据流程
```
城市列表获取 → 逐个城市爬取 → 解析页面数据 → 临时存储 → 定期保存进度
                                                          ↓
                                              爬取完成 → 保存最终CSV
```

## 注意事项

### 1. 爬取频率
- 内置100ms的爬取间隔，避免对目标网站造成压力
- 每爬取10个城市自动保存一次进度

### 2. 网络问题
- 爬虫具有基本的异常处理
- 网络超时会自动跳过当前城市并记录
- 进度保存机制确保不会丢失已爬取的数据

### 3. 数据完整性
- 中国天气网的页面结构可能会变化
- 如果页面结构变化，需要相应调整解析逻辑
- 建议定期检查爬虫是否正常工作

### 4. 历史数据
- 当前版本主要支持未来3天的天气预报
- 过去7天的历史数据获取较为复杂，需要进一步开发
- 可以根据需要扩展 `get_history_weather()` 方法

## 常见问题

### Q1: 如何修改爬取间隔？
A: 在 `weather_crawler.py` 中找到 `time.sleep(0.1)` 调用，修改参数即可（单位为秒）。

### Q2: 如何只爬取特定省份的城市？
A: 在 `get_city_list()` 方法中修改 `provinces` 列表，只保留需要的省份。

### Q3: 进度文件损坏怎么办？
A: 删除 `progress.json` 文件，爬虫会从头开始爬取。

### Q4: 如何查看爬取进度？
A: 爬虫运行时会实时显示进度：`当前进度：已完成 X/Y 个城市`。

### Q5: 支持多线程爬取吗？
A: 当前版本为单线程，确保友好爬取。如果需要多线程，可以基于当前代码扩展，但需要注意：
- 增加对目标网站的压力
- 需要更复杂的同步机制
- 可能违反网站的使用条款
